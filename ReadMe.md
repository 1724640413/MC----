## 🚀 TT语音类软件 深度完善功能开发提示词 (Markdown格式)

以下是针对 TT 语音这类以游戏和兴趣社交为核心的 App，整理的详细功能开发提示词。

---

### 一、 核心功能：语聊与游戏组队（增强版）

| 模块 | 功能提示词（需求描述） | 核心目标与技术要求 |
| :--- | :--- | :--- |
| **1.1 实时语聊** | 实现**业界领先的 QoS**，确保弱网环境下的通话质量。提供**多种降噪模式**（如环境降噪、机械键盘降噪）。 | 延迟 $\leq 100 \text{ms}$，丢包率恢复能力强。支持**3D环绕音效**，增强空间感。 |
| **1.2 快速组队** | 新增**“大神带飞”**模式，匹配高胜率、高段位用户。提供**“不语音/静音”**开关，满足不同沟通偏好。 | 匹配机制需考虑**实时在线状态**和**历史组队评分**。 |
| **1.3 悬浮窗** | 实现**游戏内悬浮球/悬浮窗**功能，允许用户在不切出游戏的情况下进行麦克风开关、音量调节和查看队友状态。 | 提升用户在游戏中的操作便利性。 |
| **1.4 游戏数据同步** | **深度集成**主流游戏 API，实现**自动同步游戏昵称、段位、常用英雄/武器**等数据。 | 确保组队信息的真实性和时效性。 |

### 二、 游戏模块（Game Integration）

| 模块 | 功能提示词（需求描述） | 核心目标与用户体验（UX） |
| :--- | :--- | :--- |
| **2.1 游戏匹配大厅** | 设置**针对不同游戏的分区大厅**（如MOBA区、射击区），每个大厅显示当前在线的组队房间和用户。 | 精准区分用户需求，提高匹配效率。 |
| **2.2 陪玩/技能服务** | 建立**“大神/陪玩”**认证体系，允许用户**发布/购买**游戏陪玩、教学等服务，并支持**按小时或按局结算**。 | 商业化变现，为用户提供专业服务。 |
| **2.3 游戏工具箱** | 为特定游戏提供**实用工具**（如游戏数据查询、出装推荐、战绩分析）的内置模块。 | 增加工具属性，提升用户在游戏中的依赖性。 |
| **2.4 赛事直播与资讯** | 内置**电竞赛事直播通道**和**游戏官方资讯聚合**模块，用户可边看直播边在房间内讨论。 | 丰富内容生态，打造一站式游戏社区。 |

### 三、 语言聊天模块（Social & IM）- K歌模式需要可上麦数量为16个！

| 模块 | 功能提示词（需求描述） | 核心目标与用户体验（UX） |
| :--- | :--- | :--- |
| **3.1 关系拓展/扩列** | **“声鉴卡”**功能，系统分析并展示用户**声线属性**。基于**兴趣标签**和**语音声线**推荐新朋友。 | 帮助用户快速找到声音和兴趣相投的伙伴。 |
| **3.2 动态社区** | 动态支持**视频话题挑战赛**。动态发布时可**@** 好友、**定位**当前位置。设置**“仅好友可见”**的隐私权限。 | 激励高质量内容产出，并保护用户隐私。 |
| **3.3 私聊与群聊** | 实现稳定的**即时通讯（IM）**功能，支持文本、图片、表情包、**短语音消息**和红包、礼物赠送。 | 满足用户一对一和小型群组的沟通需求。 |
| **3.4 好友亲密度** | 引入**“亲密度”**数值体系，通过**互送礼物、私聊时长、连续组队天数**等提升亲密度，解锁专属亲密关系标识。 | 增强用户间的羁绊和平台粘性。 |

### 四、 K歌模块（Entertainment & UGC）

| 模块 | 功能提示词（需求描述） | 核心目标与技术要求 |
| :--- | :--- | :--- |
| **4.1 实时K歌功能** | 在语聊房内加入**实时点歌、合唱、接唱**功能。提供**千万级曲库**（需版权）。 | 丰富语聊房的娱乐属性和互动性。 |
| **4.2 专业音效** | 支持用户**自定义音效预设**，包括**混响、均衡器（EQ）调节、自动降噪**和**变声效果**（如萝莉音、大叔音）。 | 提升用户演唱体验和声音表现力。 |
| **4.3 歌曲打分与榜单** | 实时显示演唱得分和音准提示。根据K歌表现设立**“麦霸榜”、“新歌榜”**等，激励用户演唱和竞争。 | 增加互动性和用户成就感。 |
| **4.4 录制与分享** | 允许用户将演唱片段**录制并保存**到个人主页，并支持**一键分享**至动态社区或外部社交平台。 | 激励用户生成内容（UGC）。 |

### 五、 房间类型（Room Categorization）

| 房间类型 | 核心主题与功能 | 适用场景与目标用户 |
| :--- | :--- | :--- |
| **5.1 游戏开黑房** | **主题：** 特定游戏（如LOL、吃鸡），显示队员位置/段位。 **功能：** 极速匹配、悬浮窗、战绩同步。 | 目标用户：有明确游戏组队需求的用户。 |
| **5.2 语音交友房** | **主题：** 闲聊、连麦、相亲、真心话大冒险。 **功能：** 随机连麦功能、麦位锁、声鉴卡展示。 | 目标用户：以社交、聊天、破冰为主要目的的用户。 |
| **5.3 娱乐K歌房** | **主题：** 音乐、K歌比赛、演唱会。 **功能：** 实时点歌系统、专业音效调节、打分和歌曲榜单。 | 目标用户：热爱音乐、喜欢展示歌喉或听歌的用户。 |
| **5.4 语音电台/FM** | **主题：** 故事、情感倾听、ASMR、脱口秀。 **功能：** 仅限房主/嘉宾发言、房间背景音循环播放。 | 目标用户：喜欢安静倾听、放松、或需要背景陪伴的用户。 |

### 六、 运营与管理（风控与数据精细化）

| 模块 | 功能提示词（需求描述） | 核心目标与风控要求 |
| :--- | :--- | :--- |
| **6.1 智能风控** | 引入**AI语音实时识别**技术，对房间内语音内容进行**低延迟审查**，及时处理**诈骗、谩骂、色情**等违规行为。 | 将违规处理时间缩短至秒级，降低平台风险。 |
| **6.2 运营活动** | 支持后台快速配置**运营活动**（如“游戏王争霸赛”、“声优招募”），包括**活动报名、进度跟踪、奖励自动发放**。 | 提高运营效率和用户参与度。 |
| **6.3 精细化数据看板** | 数据后台需提供**漏斗分析**（新用户到首付转化）、**A/B 测试工具**和**用户行为路径分析**，以便精确定位产品瓶颈。 | 实现数据驱动的产品迭代和优化。 |
| **6.4 用户认证** | 引入**实名认证/人脸识别**功能（如针对青少年防沉迷），并提供**“技能认证”**（如电竞陪玩认证）以确保服务质量。 | 确保平台合规性，提升专业服务的可信度。 |
